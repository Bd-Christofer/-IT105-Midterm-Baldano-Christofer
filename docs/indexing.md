Execution Time: 0.0008 secondsRows Examined: 10After Creating IndexSQLCREATE INDEX idx_book_title ON Books(Title);
Execution Time: 0.0003 secondsRows Examined: 2 (Only matching rows)Improvement: ~2.6x fasterWhy Is It Faster?Without Index (Full Table Scan)The database reads data like a person reading a book from page 1 to the end just to find one word.Time Complexity: $O(n)$ — If you have 10,000 rows, it performs 10,000 checks.With Index (B-Tree Search)The database uses a sorted structure (like a book's index). It jumps directly to the relevant section without looking at unrelated data.Time Complexity: $O(\log n)$ — Searching 10,000 rows takes only about 14-15 checks.Why Indexing Is Important for This SystemScalability: As the library grows from 40 books to 40,000 books, searches will remain nearly instant.User Experience: Students shouldn't have to wait for the system to "load" when just checking if a book is in stock.Efficiency: Reduces the workload on the server, which is critical for systems handling many simultaneous users.
